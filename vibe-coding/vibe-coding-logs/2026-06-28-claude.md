**On format**, my honest recommendation is to reconsider YAML for this specific case:

- YAML requires PyYAML, a third-party dependency. So far the app needs nothing beyond PyGObject, and one of its nice properties is zero pip dependencies. Adding PyYAML just for a tiny config file is a poor trade.
- For your two settings (a font string and a list of paths), **TOML** is arguably a better fit and is in the standard library as `tomllib` (Python 3.11+) for reading. The catch: `tomllib` is read-only — there's no stdlib TOML *writer* — so you'd need `tomli-w` to write, which reintroduces a dependency.
- **JSON** is in the stdlib (read and write), human-editable enough for this, and supports nested dicts for future growth. It's less pretty than YAML for hand-editing (no comments), but it's dependency-free both ways.

So the real choice is: dependency-free JSON, or nicer-to-hand-edit YAML at the cost of a dependency.

Given your stated priorities (human-editable, git-trackable, room for nested dicts later), and that you explicitly raised YAML, here's how I'd weigh it:

- If "no new dependencies" matters to you → **JSON**.
- If hand-editability and comments matter more → **YAML** with PyYAML, gracefully degrading if it's not installed.

My slight preference is JSON here, purely to keep the dependency story clean, with the config module written so the format is swappable later. But it's genuinely close, and your reasoning for YAML is sound.

**Card view toggle** (off by default), added to the toolbar after Slugify:
- `model.py` gained `first_body_line(note)` (first non-blank line after a leading heading, skipping the heading; falls back to the first non-blank line when there's no heading) and `format_mtime(note)` (`YYYY-MM-DD HH:MM`).
- The note-list store gained a 4th **markup** column, and the cell renderer now binds to markup instead of plain text. `_note_markup(note)` builds it: in list view just the (escaped) title as before; in card view the title in **bold**, then the last-modified date and the first body line in a smaller, regular-weight grey font — exactly the three-line layout you specified.
- `on_toggle_card_view` flips the mode and reloads the list, preserving the current selection. All dynamic text is XML-escaped so note content can't break the markup.

**New note button no longer highlighted on startup** — initial keyboard focus is now placed on the sidebar (`set_focus(self.sidebar_view)`), so no toolbar button shows a focus ring when the window opens.

Verified with a new card-view stub (plain vs card markup, clean toggle back, escaping of special characters) plus the four existing suites (round-2 features, preview, search, read-only/tree/tab-nav) — all green.

A couple of things to eyeball on your real MATE desktop, since I can't render GTK here:
- The Card view button uses the stock `view-list-details-symbolic` icon — swap it if you'd prefer something more card-like (e.g. `view-grid-symbolic` if your theme has it).
- Card rows are taller than list rows, so confirm the spacing and the small grey date/snippet lines look right against your theme, and that long first-lines ellipsize acceptably.
